Команды организованы через классы


Если кратко
Команды USER, NICK и PASS организованы через классы, 
каждая проверяет состояние клиента перед выполнением.
Они добавляются в сервер в initializeCommands(), 
вызываются через handleCommand(), 
и отправляют клиенту send()-сообщения с кодами IRC.


Если длинно и подробно
Как обрабатываются?

1. Сервер получает команду от клиента.
2. Определяеся какая команда (NICK/USER/PASS).
3. Находит соответствующий класс (NickCommand, UserCommand, PassCommand).
4. Вызываеся метод execute(client, args), передавая клиента и аргументы команды.
5. В классе команды проверяется корректность выполнения:
Есть ли параметры?
Все ли ок с порядком комманд?
И модет ли выполнится команда в текущем состоянии клиента?
6. Если команда выполнена успешно, клиенту отправляется ответ. Ну или ошибка, если не ок.

PASS (аутентификация)
Это первое что должен сделать Клиент, чтобы продолжить.

NICK (установка/изменение никнейма)
Никнейм можно менять, но клиент должен быть аутентифицирован.

USER (установка имени пользователя)
USER можно установить только один раз.

Если клиент нарушает порядок 
(USER без NICK или команды без PASS), сервер возвращает ошибку.


-------
Команды проверяют состояние клиента

Каждая команда перед выполнением проверяет:

Прошла ли аутентификация (PASS)?
Установлены ли NICK и USER?
Есть ли ошибки в аргументах?

 --- Ошибки передаются клиенту через send() ---
